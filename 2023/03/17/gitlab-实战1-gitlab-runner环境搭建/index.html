gitlab-实战1-gitlab-runner环境搭建

发表于 2023-03-17 分类于 devOps

一、前言

文章主要记录容器化环境下的gitlab-runner的安装和使用，即执行器采用docker。
由于在实战过程中，搜集的各方面的资料比较零散，所以对这些资料进行整理，并在过程中遇到的问题进行记录。

二、配置

  1. gitlab环境
  2. rancher环境或者docker环境（以rancher环境为例）
  3. harbor环境

三、实战目标

将gitlab源码通过.gitlab-ci.yml触发gitlab-runner进行打包，发布镜像，推送到harbor仓库，执行容器化部署。

四、安装gitlab-runner

镜像: gitlab/gitlab-runner:v12.10.3
端口映射: 无
数据卷映射:

  • /var/run/docker.sock:/var/run/docker.sock(用于docker作为runner的执行器时，能够能够操作宿主机的docker)
  • /etc/gitlab-runner:/etc/gitlab-runner(用于runner配置持久化)
  • /etc/gitlab-runner-docker:/root/.docker/(用于私有仓库镜像拉取用户鉴权)
  • 其他需要映射到宿主机目录的共享数据，比如maven的仓库等等。

4.1 docker安装

docker run -d –name gitlab-runner -v /var/run/docker.sock:/var/run/docker.sock -v /etc/gitlab-runner:/etc/gitlab-runner -v /etc/gitlab-runner-docker:/root/.docker/ –restart always gitlab/gitlab-runner:v12.10.3

4.2 rancher安装
对应手动配置数据卷映射即可

五、gitlab-runner注册

1.使用root用户进行操作注册
root@dj:/# gitlab-runner register

2.根据提示输入相关配置

Please enter the gitlab-ci coordinator URL (e.g. https://gitlab.com/):
（输入GitLab实例URL）
Please enter the gitlab-ci token for this runner:
（输入runner注册令牌）
Please enter the gitlab-ci description for this runner:
（输入说明）
Please enter the gitlab-ci tags for this runner (comma separated):
（输入runner标签 可在gitlab的UI页面修改）
Registering runner… succeeded runner=JbkzB1Zy
Please enter the executor: docker-ssh, parallels, docker+machine, kubernetes, custom, docker, shell, ssh, virtualbox, docker-ssh+machine:
（输入runner脚本执行方式，此处选择docker, 并在后续设置一个默认镜像）
注册成功后在gitlab的相应组和项目页面： 设置->CI/CD->Runners页面会出现一个刚注册的Runner和tags标签，tags标签为刚才输入的tag标签内容

3.重启gitlab-runner
root@dj:/# gitlab-runner restart

4.查看已注册成功的Runner列表
root@dj:/# gitlab-runner list

九、总结

  1. 在使用docker来部署gitlab-runner的过程是很漫长的，因为网上的资料比较零散， 包括使用docker作为gitlab-runner的执行器，之后的各种命令都是依托基础镜像来完成，所以后续有重新基于这些基础镜像来添加公司的一些私有信息来构建成公司的基础镜像。
  2. 目前的实战停留在了rancher部署的这个环节，基础镜像已经构建完成，后续就是触发rancher的容器编排部署，后续再跟进。
  3. 总体来说，容器化gitlab-runner存在一定的优势，gitlab-runner的部署工作完成后，由于后续的持续集成涉及到的环境，平台都是基于镜像来完成的。所以，只要有对应的基础镜像，无需在服务器部署对应的环境，直接调取基础镜像来进行项目打包和推送。
